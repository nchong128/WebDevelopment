Hash Tables
Friday, 28 September 2018

Learn about hash functions which allow the entire premise of hash tables to work!

Definition

A hash function is any function that can be used to map data of arbitrary size to data of a fixed size.



A desirable hash function
  • Fast and still linear in the size of the key
    • A slow one will degrade performance
  • Minimises collisions (two keys mapped to the same position)
    • Don't use non-data or anything in our key that does not pose any useful information (e.g. no capital letters)
    • Use all elements (or a reasonable subset - odd/even positions)
    • Use the position of each element
    • Avoid common factors
  • Deterministic (same value, same input)
  • Returns a value within the table range, so each value corresponds to an index in the table
